# ETF 动量轮动系统 — 启动与部署文档

> 本文档面向开发者与运维人员，说明如何在本地开发环境及生产环境启动、部署 ETF 动量轮动策略系统。

---

## 一、环境要求

### 1.1 必需环境

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 17+ | 后端运行环境 |
| Maven | 3.8+ | 后端构建工具 |
| MySQL | 8.0+ | 数据持久化 |
| Node.js | 18+ | 前端构建与开发 |
| npm / pnpm | npm 9+ / pnpm 8+ | 前端包管理 |

### 1.2 推荐开发环境

- **操作系统**：macOS / Linux / Windows with WSL2
- **IDE**：IntelliJ IDEA（后端）、VS Code（前端）
- **浏览器**：Chrome 90+ / Edge 90+ / Safari 15+

### 1.3 端口规划

| 服务 | 默认端口 | 说明 |
|------|---------|------|
| Spring Boot 后端 | 8080 | 提供 REST API 与静态资源 |
| Vue 前端开发服务器 | 5173 | 开发模式热更新 |

## 二、后端启动

### 2.1 创建数据库

使用 MySQL 客户端执行初始化脚本：

```bash
# 登录 MySQL 后创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS momentum_strategy DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 执行表结构初始化
mysql -u root -p momentum_strategy < src/main/resources/db/migration/V1__init_schema.sql

# 执行基础数据初始化（候选 ETF、策略配置）
mysql -u root -p momentum_strategy < src/main/resources/db/migration/V2__init_data.sql
```

> 如果数据库中已有历史数据，后续迁移脚本按版本顺序执行：V3、V4、V5……

### 2.2 配置数据库连接

编辑 `src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/momentum_strategy?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: 你的密码
```

### 2.3 编译与测试

```bash
# 编译并运行单元测试
mvn clean test

# 仅编译
mvn clean compile -DskipTests
```

### 2.4 启动后端服务

#### 方式一：Maven 插件启动（开发推荐）

```bash
mvn spring-boot:run
```

#### 方式二：打包后启动

```bash
mvn clean package -DskipTests
java -jar target/fund-momentum-strategy-1.0.0.jar
```

#### 方式三：指定端口启动

```bash
java -jar target/fund-momentum-strategy-1.0.0.jar --server.port=8081
```

### 2.5 验证后端启动

访问以下地址确认后端正常运行：

| 地址 | 预期结果 |
|------|---------|
| http://localhost:8080/api/funds | 返回候选 ETF JSON 列表 |
| http://localhost:8080/dashboard/api/data | 返回策略看板数据 |
| http://localhost:8080/actuator/health | 健康检查（如启用） |

## 三、前端启动

### 3.1 安装依赖

进入前端工程目录：

```bash
cd frontend

# 使用 npm
npm install

# 或使用 pnpm（需先安装 pnpm）
pnpm install
```

### 3.2 开发模式

```bash
npm run dev
```

开发服务器默认运行在 http://localhost:5173，API 请求会通过 Vite 代理自动转发到 http://localhost:8080。

> 确保后端服务已启动，否则页面数据无法加载。

### 3.3 生产构建

```bash
npm run build
```

构建产物会自动输出到 `src/main/resources/static/`，Spring Boot 启动后会自动托管这些静态资源。

### 3.4 前端生产验证

启动后端后访问：

- http://localhost:8080 —— 新版 Vue 前端首页
- http://localhost:8080/dashboard —— 策略看板
- http://localhost:8080/rank —— 动量排名

## 四、首次使用数据初始化

### 4.1 同步基金净值

首次启动后，建议同步 1 年以上的历史净值数据：

```bash
# 同步全部候选 ETF
curl -X POST "http://localhost:8080/api/funds/sync?startDate=2023-01-01&endDate=2024-12-31"

# 或同步单只基金
curl -X POST "http://localhost:8080/api/funds/510300/sync?startDate=2023-01-01&endDate=2024-12-31"
```

也可通过前端「基金池」页面的「同步净值」按钮操作。

### 4.2 同步市场数据

```bash
curl -X POST "http://localhost:8080/api/market-data/sync"
```

> 市场数据包含 A 股成交量与资金流向，是策略看板的重要数据源。

### 4.3 执行策略

```bash
curl -X POST "http://localhost:8080/api/strategy/execute?tradeDate=2024-01-05"
```

或在前端「策略执行」页面点击「执行策略」。

### 4.4 运行回测

```bash
curl -X POST http://localhost:8080/api/backtest/run \
  -H "Content-Type: application/json" \
  -d '{
    "startDate": "2023-01-01",
    "endDate": "2024-01-01"
  }'
```

## 五、生产部署

### 5.1 单体部署（推荐）

将前端构建产物放入后端静态资源目录，打包为单一 JAR：

```bash
# 1. 构建前端
cd frontend
npm install
npm run build
cd ..

# 2. 构建后端
mvn clean package -DskipTests

# 3. 部署运行
java -jar target/fund-momentum-strategy-1.0.0.jar
```

### 5.2 前后端分离部署

#### 后端

```bash
mvn clean package -DskipTests
java -jar target/fund-momentum-strategy-1.0.0.jar
```

#### 前端

使用 Nginx 托管 `frontend/dist/`：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://localhost:8080/api/;
    }

    location /dashboard/ {
        proxy_pass http://localhost:8080/dashboard/;
    }
}
```

### 5.3 使用 Docker 部署（可选）

```dockerfile
# Dockerfile
FROM eclipse-temurin:17-jdk
COPY target/fund-momentum-strategy-1.0.0.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "/app.jar"]
```

```bash
mvn clean package -DskipTests
docker build -t fund-momentum:latest .
docker run -d -p 8080:8080 --name fund-momentum fund-momentum:latest
```

## 六、定时任务

系统内置以下定时任务：

| 任务 | 表达式 | 说明 |
|------|--------|------|
| 净值同步 | `0 35 15 ? * MON-FRI` | 工作日 15:35 同步近 6 个月净值 |
| 策略执行 | `0 35 15 ? * FRI` | 每周五 15:35 执行策略计算 |
| 市场数据同步 | `0 35 16 ? * MON-FRI` | 工作日 16:35 同步市场数据 |

> 定时任务默认启用，可通过 Spring 的 `@Scheduled` 配置或前端「任务调度」页面手动触发。

## 七、常见问题排查

### 7.1 前端无法访问后端 API

**现象**：页面加载空白，控制台报 404/500。

**排查**：
1. 确认后端已启动并监听 8080 端口
2. 开发模式下，Vite 代理已配置 `vite.config.ts`
3. 生产模式下，确认前端构建产物已放入 `src/main/resources/static/`

### 7.2 策略执行后没有持仓建议

**可能原因**：
- 净值数据不足，无法计算均线/波动率
- 当前市场环境弱势，积极型策略清仓
- 候选 ETF 均未通过动量/质量/均线过滤

**解决**：先执行净值同步，确保数据完整。

### 7.3 数据库连接失败

**排查**：
1. 确认 MySQL 服务已启动
2. 检查 `application.yml` 中的用户名、密码、数据库名
3. 确认 MySQL 允许远程/本地连接
4. 首次启动需执行 `V1__init_schema.sql` 和 `V2__init_data.sql`

### 7.4 前端构建失败

**常见原因**：
- Node.js 版本过低（需 ≥ 18）
- 网络问题导致依赖下载失败
- TypeScript 类型错误

**解决**：
```bash
# 清理依赖后重新安装
rm -rf frontend/node_modules frontend/package-lock.json
cd frontend
npm install
npm run build
```

### 7.5 静态资源 404

**排查**：
1. 确认已执行 `npm run build`
2. 确认 `src/main/resources/static/` 下存在 `index.html`
3. 确认 `SpaFallbackController` 已编译进 JAR
4. 检查是否有其他 Controller 占用了前端路由路径
