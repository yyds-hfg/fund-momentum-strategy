## 一、文档概述

### 1.1 编写目的

本文档对项目现有设计文档进行统一梳理，并结合以下三个维度规划未来迭代方向：

1. **前端工程化升级**：以 [vue-element-admin](https://github.com/PanJiaChen/vue-element-admin) 为基础，构建独立、可扩展的策略管理后台。
2. **策略能力扩展**：在现有“双动量 + 上涨质量 + 低波 + 均线 + 大盘过滤”核心逻辑之上，增加多策略、多因子、多市场支持。
3. **AI Agent 生态融合**：结合 Kimi Claw（云端持续 AI 代理）与 Kimi Work（本地/桌面通用 Agent），将策略系统从“计算工具”升级为“自动化投研助手”。

## 二、现有设计整理

### 2.1 核心策略逻辑

```
不买跌的（绝对动量 > 0）
  → 涨得稳的（上涨天数占比 > 50%）
    → 买最强的（相对动量排序）
      → 稳的多买（低波加权）
        → 趋势确认（收盘价 > N 日均线）
          → 大盘环境调整（中证 A500 120 日均线）
```

### 2.2 双策略混用参数

| 参数 | 平衡型 (BALANCED) | 积极型 (ACTIVE) |
|------|-------------------|-----------------|
| 资金占比 | 70% | 30% |
| 短期动量窗口 | 10 日 | 5 日 |
| 长期动量窗口 | 20 日 | 10 日 |
| 上涨天数占比阈值 | > 50% | > 50% |
| 均线周期 | 60 日 | 20 日 |
| 波动率窗口 | 20 日 | 10 日 |
| 最大持仓 | 3 只 | 2 只 |
| 单只权重上限 | 40% | 50% |
| 调仓频率 | 周度 | 周度 |
| 冷静期 | 3 个交易日 | 3 个交易日 |

### 2.3 风控规则

| 规则 | 触发条件 | 操作 |
|------|---------|------|
| 止损 | 单只 ETF 自进入建议持仓以来，最高收盘净值回撤 > 8% | 无条件清仓 |
| 单只上限 | 权重超过策略配置上限 | 截断并二次归一化 |
| 冷静期 | 新买入未满 3 个交易日 | 不换仓（止损例外） |
| 大盘仓位 | 中证 A500 收盘价 ≤ 120 日均线 | 平衡型仓位减半，积极型清仓 |

### 2.4 DDD 四层架构

```text
┌─────────────────────────────────────────────┐
│  接口适配层 (Adapter)                       │
│  Controller / 定时任务 / 外部数据爬虫适配器   │
├─────────────────────────────────────────────┤
│  应用层 (Application)                       │
│  AppService：编排领域服务、DTO、事务边界      │
├─────────────────────────────────────────────┤
│  领域层 (Domain) ★ 核心                     │
│  聚合根、值对象、领域服务、仓储接口、领域事件 │
├─────────────────────────────────────────────┤
│  基础设施层 (Infrastructure)                │
│  仓储实现、Mapper、数据库、外部 API 客户端   │
└─────────────────────────────────────────────┘
```

## 三、前端重构：基于 vue-element-admin

### 3.1 为什么选 vue-element-admin

[vue-element-admin](https://github.com/PanJiaChen/vue-element-admin) 是 GitHub 上最受欢迎的开源中后台前端解决方案之一（88k+ stars），具备以下与本项目高度匹配的能力：

| 能力 | 对应本项目需求 |
|------|---------------|
| 登录/权限认证 | 多角色：管理员、研究员、只读用户 |
| 动态侧边栏 + 多级路由 | 基金池、策略配置、看板、回测、报告、系统管理 |
| Dashboard 模板 | 策略看板首页（指标卡 + ECharts） |
| ECharts 集成 | 成交量、资金流向、回测净值曲线、回撤曲线 |
| 动态表格/拖拽表格 | 基金池管理、持仓推荐、回测记录 |
| Excel 导入导出 | 基金批量导入、周报/回测结果导出 |
| 多环境构建 | dev / sit / stage / prod |
| i18n 国际化 | 后续支持中英文切换 |
| Mock 数据 | 前后端并行开发 |

> **重要提示**：vue-element-admin 原版基于 Vue 2 + Element UI。考虑到 Vue 2 已于 2023 年底停止维护，**建议采用其 Vue 3 精神续作**（如 [vue3-element-admin](https://github.com/youlaitech/vue3-element-admin)、Vue-Vben-Admin 或 Vue Element Plus Admin），以获得更长的技术生命周期和更好的 TypeScript/Vite 体验。若团队熟悉 Vue 2，原版仍可短期使用，但需规划迁移。

### 3.4 前端实施建议

1. **优先迁移策略看板**：现有 Thymeleaf 看板已具备数据接口，迁移成本最低，效果最明显。
2. **引入 TypeScript**：提升金融数据模型的类型安全。
3. **统一图表组件**：封装 ECharts 组件，统一配色、tooltip、响应式。
4. **Mock 数据先行**：前端团队可并行开发，不受后端进度影响。
5. **权限设计**：
   - `admin`：系统配置、用户管理、策略参数修改
   - `researcher`：执行策略、运行回测、查看全部数据
   - `viewer`：只读查看看板和报告

### 4.3 具体可扩展方向

#### 方向一：AI 驱动的策略信号解读

**现状**：系统输出“买入 A 30%、B 20%”，但不解释为什么。

**扩展**：
- 调用 Kimi API 对每次策略结果生成自然语言解读：
  - 当前市场状态（强势/弱势）
  - 入选 ETF 的动量、上涨质量、均线状态
  - 调仓原因（新增/减持/止损/冷静期）
  - 风险提示
- 在看板“推荐持仓”旁增加“AI 解读”按钮。

**价值**：降低策略使用门槛，让非量化背景的投资者也能理解。

#### 方向二：定时自动化研报生成

**现状**：周报需要手动触发或定时生成固定模板。

**扩展**：
- 使用 **Kimi Claw 定时任务**，每周五收盘后自动执行：
  1. 调用后端 `/api/strategy/execute`
  2. 抓取本周市场要闻、行业资金流向
  3. 生成 Markdown/PDF 周报
  4. 推送至飞书/钉钉/邮件/企业微信

**价值**：实现“策略计算 → 市场解读 → 报告生成 → 推送”全自动化。

#### 方向三：舆情与资金面增强因子

**现状**：策略仅使用价格和成交量数据。

**扩展**：
- 通过 **Kimi Work WebBridge** 或后端爬虫，抓取：
  - 财经新闻、行业政策、券商研报
  - ETF 资金流向、北向资金、融资融券
  - 社交媒体情绪（雪球、东方财富股吧）
- 使用 Kimi API 对文本进行情绪分析、主题提取，生成**舆情因子**。
- 将舆情因子作为辅助过滤条件：
  - 负面舆情占比高 → 降低该 ETF 权重或剔除
  - 政策利好主题 → 提高相关行业 ETF 关注度

**价值**：从纯价格动量扩展到“价格 + 资金 + 舆情”多因子模型。

#### 方向四：智能回测与参数优化

**现状**：回测参数固定，依赖人工经验。

**扩展**：
- 使用 Kimi Agent 对策略参数进行**网格搜索 / 遗传算法优化**：
  - 动量窗口、均线周期、上涨阈值、权重上限
  - 目标：最大化夏普比率或卡玛比率
- 生成参数敏感性分析报告。
- 通过 Kimi Work 在本地运行 Python 脚本，调用后端回测接口批量执行。

**价值**：提升策略参数的科学性，减少人工试错。

#### 方向五：真实持仓跟踪与交易指令生成

**现状**：仅生成建议持仓，用户需手动在券商执行。

**扩展**：
- 新增 `RealPosition` 聚合根和 `TradeOrder` 实体。
- 用户上传/录入真实持仓后，系统自动对比建议持仓：
  - 生成“买入/卖出/持有”交易指令清单
  - 计算预计交易成本、冲击成本
  - 跟踪执行后的实际收益与策略收益的差异
- 结合 Kimi Work 读取本地券商对账单（Excel/PDF），自动更新真实持仓。

**价值**：打通“建议 → 执行 → 跟踪”闭环。

#### 方向六：多市场与多资产扩展

**现状**：仅支持 A 股 ETF。

**扩展**：
- 增加港股 ETF、美股 ETF、REITs、黄金、债券等资产类别。
- 每个资产类别配置独立的策略参数和数据源。
- 通过 Kimi Work 抓取海外市场数据（如 Yahoo Finance、Bloomberg）。

**价值**：构建跨市场资产配置能力，分散单一市场风险。

#### 方向七：风险预警与异常监控

**现状**：止损在策略执行时判断，无实时盘中监控。

**扩展**：
- Kimi Claw 定时任务每 30 分钟检查：
  - 持仓 ETF 是否触发 8% 回撤止损
  - 大盘是否由强转弱
  - 净值数据是否缺失
- 触发异常时，通过 IM/Webhook 推送告警。

**价值**：提升风控响应速度，避免人工盯盘遗漏。

#### 方向八：策略知识库与问答助手

**现状**：策略逻辑分散在文档中，查询不便。

**扩展**：
- 将项目所有 Markdown 文档、代码注释、历史报告导入 Kimi 知识库。
- 在前端提供“策略问答助手”：
  - “为什么今天没有持仓？”
  - “平衡型和积极型的区别是什么？”
  - “上周调仓了哪些基金？”
- 助手直接基于知识库和实时数据回答。

**价值**：降低新用户学习成本，提升系统可维护性。

## 六、附录

### 6.1 技术选型建议

| 层级 | 建议方案 | 备选方案 |
|------|---------|---------|
| 前端框架 | Vue 3 + Vite | Vue 2 + vue-element-admin（原版） |
| UI 组件库 | Element Plus | Ant Design Vue / Naive UI |
| 状态管理 | Pinia | Vuex 4 |
| HTTP 客户端 | Axios | Fetch |
| 图表 | ECharts 5 | AntV G2Plot |
| 后端安全 | Spring Security + JWT | 自定义 Filter + JWT |
| AI 接入 | Kimi Platform API | OpenAI / Claude / DeepSeek API |
| 定时任务 | Kimi Claw Cron + 后端 Spring Scheduler | Quartz |
| 报告生成 | Kimi Work Skill | 后端 poi-tl / iText |

### 6.2 风险提示

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| vue-element-admin 原版基于 Vue 2，生命周期有限 | 中 | 优先选择 Vue 3 版本 |
| 东方财富/新浪接口不稳定或限流 | 高 | 多源抓取 + 本地缓存 + 降级 |
| Kimi API 成本随调用量上升 | 中 | 缓存解读结果、控制调用频率 |
| Kimi Work/Claw 处于快速迭代期 | 中 | 封装抽象层，便于切换底层实现 |
| 金融数据合规性 | 高 | 仅使用公开数据，不存储用户敏感持仓 |
| 策略收益不及预期 | 高 | 明确免责声明，强调系统仅生成建议 |
